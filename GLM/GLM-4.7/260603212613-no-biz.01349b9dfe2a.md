# Feature Name:
- Hệ thống Quản lý Tệp Tích Lưu (Google Drive Clone)

# Description:
- Ứng dụng web cho phép người dùng tải lên, tải xuống, tổ chức và chia sẻ tệp qua trình duyệt.

# Actors:
- Primary: End User (Người dùng)
- Secondary: System Admin (Quản trị hệ thống)

# Business Goal:
- Tạo trung tâm lưu trữ tệp, hỗ trợ collaboration, và giảm nhu cầu lưu trữ vật lý.

# Functional Requirements:
- Upload/Download tệp (Drag & Drop, Button).
- Quản lý thư mục (Tạo, Xóa, Đổi tên, Di chuyển).
- Tìm kiếm tệp (Search bar).
- Chia sẻ tệp (Public/Private link, quyền truy cập).
- Xem trước tệp (Preview: Image, Video, Text).
- Breadcrumb navigation.

# Critical Considerations:
1. Security:
   - Kiểm tra loại tệp (file type validation) để ngăn chặn code execution.
   - Mã hóa dữ liệu khi truyền qua mạng.
   - Xác quyền (Authorization).
2. Data Integrity:
   - Xử lý xung đột khi tải lên tệp trùng tên.
3. Performance:
   - Upload progress bar cho tệp lớn.
   - Caching thumbnails.
4. UX:
   - Trạng thái loading, empty state, error message rõ ràng.
   - Breadcrumb navigation.
5. Edge Cases:
   - Tên tệp có ký tự đặc biệt (Unicode).
   - Tệp rất lớn (>1GB).
   - Tệp corrupt.

# Risks:
- Chi phí lưu trữ (Storage cost) tăng theo thời gian.
- Lỗi mất dữ liệu (Data loss) nếu crash hệ thống.
- Tấn công bảo mật từ tệp tải lên (Malicious file upload).

# Non-functional Requirements:
- Scalability: Hệ thống phải chịu tải nhiều người dùng cùng lúc.
- API Rate Limiting: Giới hạn số request mỗi giây.
- Logging: Ghi log tất cả hành động người dùng.

# Open Questions:
- Backend storage là gì? (AWS S3, Local File System, Cloud SQL?)
- Max file size cho phép?
- Có cần versioning (phiên bản) tệp không?
- Có cần real-time collaboration (đồng nhập) không?
